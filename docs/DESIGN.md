# 제품 · UX 설계

## 문제

친구들끼리 롤 랭크(또는 자유랭) 5인 파티를 카카오톡 오픈채팅방에서 모을 때,
누가 어느 라인을 갈지 텍스트로 주고받으면 겹치고 헷갈린다.
"탑 나요", "미드요"가 스크롤에 묻히고, 중복·공백이 생긴다.

## 해결

**포지션 보드**를 링크로 공유한다. 각자 링크를 열어 자기 라인을 누르면,
모두의 화면에서 즉시 그 자리가 채워진다. 5칸이 다 차면 모집 완료.

> 카카오톡 오픈채팅방에는 공식 봇/자동응답 API가 없다.
> 비공식 안드로이드 알림봇은 이용약관 위반·계정 정지 위험이 있어 배제했다.
> 대신 **공유 링크(웹앱)** 방식 — 합법적이고 기기 제약이 없다.

## 포지션

| 코드 | 한글 | 영문 |
|------|------|------|
| `TOP` | 탑 | TOP |
| `JGL` | 정글 | JUNGLE |
| `MID` | 미드 | MID |
| `ADC` | 원딜 | BOT · ADC |
| `SUP` | 서폿 | SUPPORT |

단일 출처: `src/domain/positions.ts`.

## 사용자 흐름

1. **방장**이 `?room=우리방` 링크를 열고 오픈채팅방에 붙여넣는다.
2. 참가자가 링크를 열고 **닉네임**을 입력한다(마지막 닉네임은 브라우저에 기억됨).
3. 원하는 **라인을 클릭** → 즉시 그 자리를 차지. 모두의 화면이 실시간 갱신.
4. 마음이 바뀌면 다른 라인을 클릭 → 자동 이동. `✕`로 자리 비우기.
5. **5/5**가 되면 카운터가 강조되어 모집 완료를 알림.

### 규칙

- 한 사람은 한 자리만. 다른 라인 클릭 시 이전 자리를 비우고 이동.
- 이미 찬 자리는 남이 못 뺏음(`POSITION_TAKEN`).
- 본인이 앉은 자리만 비울 수 있음(`NOT_SEAT_OWNER`).
- 닉네임은 1~16자(공백 트림).
- 연결이 끊기면(창 닫기 등) 그 사람 자리는 자동 해제.

## 방(Room) 개념

오픈채팅방 하나 = 방 하나. 방은 `?room=<코드>`로 구분되며, 서로 완전히 격리된다.
`새 방` 버튼은 무작위 6자 코드를 만들어 새 링크를 준다.

## 비주얼 방향

**헥스테크(Hextech)** — LoL 클라이언트의 정체성: 딥 네이비 그라운드, 골드 라인,
차지된 자리는 틸(teal)로 강조. 라이트/다크 테마 모두 토큰 기반으로 설계.

- 위계: 헤드라인 대비 · 라인 카드의 좌측 컬러 바(빈 자리=뮤트, 찬 자리=틸)
- 상태: 빈 자리(점선 "＋ 선택") vs 찬 자리(닉네임 + 내 자리엔 해제 버튼)
- 접근성: `aria-label`, 키보드 포커스 링, `prefers-reduced-motion` 존중,
  `aria-live`로 인원수 변화 안내
- 성능: 컴포지터 친화 속성(transform/opacity)만 애니메이션, 번들러·외부 폰트 없음

## 카카오톡과의 접점

- 채팅창에 링크를 붙이면 Open Graph 메타(제목/설명)로 카드 미리보기가 뜬다.
- 자동 닉네임 불러오기(카카오 로그인)는 **오픈채팅 별명과 실제 계정 닉네임이 달라**
  혼란을 주므로 채택하지 않고, 직접 입력 방식을 택했다. (ROADMAP의 선택지로 남김)
