# 로드맵 · 보완점

MVP는 완성(5포지션 실시간 모집, 방 격리, 연결 종료 정리, 80%+ 테스트).
아래는 우선순위순 개선 과제 — 어시스턴트가 설계 중 발견해 추가한 항목 포함.

## 1. 소유권 안정화 (재연결 대응) — 우선순위 높음
지금 자리 소유는 **소켓 id**에 묶여 있다. 폰이 잠기거나 네트워크가 끊겨 재연결하면
새 소켓 id가 발급되어 **자기 자리를 잃거나 해제 권한을 잃는다.**
→ localStorage에 안정적인 **클라이언트 토큰(UUID)**을 저장하고 그걸 `ownerId`로 사용.
   Socket.IO **connection state recovery** 또는 재연결 시 자동 재차지를 결합.
   (단일 기기·연속 세션에선 문제 없으나, 견고한 제품엔 필수.)

## 2. 영속성 & 수평 확장
인메모리 상태는 서버 재시작 시 사라지고 다중 인스턴스에서 공유되지 않는다.
→ `RoomRegistry`를 인터페이스로 추출하고 **Redis/SQLite** 구현 주입.
   다중 인스턴스는 **Socket.IO Redis 어댑터**로 브로드캐스트 팬아웃.

## 3. 방장/설정 UI
`RoomSettings(title/time/tier/queue)`는 모델·DTO엔 있으나 편집 UI가 없다.
→ 방장이 시간·티어·큐·제목을 편집(그 방의 host 소켓만). 방장 위임/표시.

## 4. 모집 완료 상태 & 잠금
5/5가 되면 "모집 완료"로 전환하고 새 차지를 잠그는 옵션.

## 5. 프레즌스(접속 인원)
방에 몇 명이 보고 있는지 실시간 표시.

## 6. 남용 방지
방당/소켓당 **레이트 리밋**, 닉네임 금칙어 필터(선택), 방 코드 추측 방지.

## 7. 방 수명(TTL)
비어 있는 방을 일정 시간 후 정리(메모리 회수).

## 8. 카카오톡 링크 카드(OG 이미지)
현재 OG 제목/설명만 있음. **OG 이미지**를 추가해 채팅창 미리보기를 예쁘게.

## 9. 배포 가이드 & 보안 헤더
Render/Railway/Fly 배포 문서, HTTPS 뒤 CSP·HSTS·`X-Content-Type-Options` 등.

## 10. 접근성·i18n·크로스브라우저
자동 접근성 검사, 키보드 내비게이션·리듀스드모션 검증, 다국어(선택),
Chrome/Firefox/Safari 스모크.

## 11. (선택) 라이엇 API 연동
소환사명 검증·티어 자동 표기. 키 관리·레이트 리밋 고려 필요.

## 12. (대안 검토) Supabase/Firebase 백엔드
자체 서버 운영을 피하고 싶다면 Supabase Realtime으로 전송 계층 교체 가능.
`domain/`은 그대로 재사용된다(교체 지점은 `server/`).
