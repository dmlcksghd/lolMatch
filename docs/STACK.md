# 기술 스택 — 무엇으로 만들었나

## 한 줄 요약
**TypeScript(서버) + JavaScript(브라우저)** · Node.js · Express · **Socket.IO(실시간)** · Vitest ·
프레임워크·번들러 없는 순수 프론트엔드 · 인메모리 저장 · Render 배포.

## 언어
- **TypeScript** — 서버와 핵심 도메인 로직(타입 안전).
- **JavaScript** — 브라우저 클라이언트(`public/app.js`).
- 둘 다 JS 계열이라 사실상 한 언어권.

## 런타임 · 서버
| 요소 | 역할 |
|------|------|
| **Node.js 20+** | 실행 런타임 |
| **Express** | HTTP 서버 — 정적 파일 제공 + `/healthz` 헬스체크 |
| **Socket.IO** | 실시간 양방향 통신(WebSocket). 자리·설정 변경을 방 전체에 즉시 브로드캐스트 |

## 프론트엔드(브라우저)
- **순수 HTML / CSS / JavaScript** — React 등 프레임워크·번들러 없음.
- Socket.IO 클라이언트는 서버가 `/socket.io/socket.io.js`로 자동 제공.
- CSS: 디자인 토큰(커스텀 프로퍼티), 라이트/다크, 헥스테크(LoL) 스타일, 접근성 고려.

## 저장 · 테스트 · 빌드 · 배포
| 요소 | 내용 |
|------|------|
| 저장 | **인메모리**(서버 메모리). DB 없음 — 킵얼라이브로 유지, 재배포 시 초기화 |
| 테스트 | **Vitest** — 단위(도메인) + 통합(실서버 소켓), 52개 |
| 실행 | **tsx**로 TS를 컴파일 없이 실행. `tsc`는 타입체크용 |
| 배포 | **Render**(무료) + `render.yaml`. CI: **GitHub Actions**(타입체크·테스트) + 킵얼라이브 |

## 구조 (3계층)
```
브라우저 (HTML/CSS/JS + Socket.IO client)
    ↕  WebSocket
Node.js 서버 (Express + Socket.IO)        ← TypeScript
    ↕
도메인 로직 (순수 TS: 포지션·티어·큐·게임 수명)  ← 프레임워크 무관, 테스트 핵심
    ↕
인메모리 저장 (RoomRegistry)
```

## 왜 이렇게
- **한 언어(JS/TS)** 로 서버·클라이언트를 통일 → 배우고 유지하기 쉬움.
- **Socket.IO** 는 실시간 + 재연결 + 룸을 내장 → 직접 구현할 게 적음.
- **번들러 없는 프론트** → 마이크로사이트 규모라 빌드 파이프라인이 불필요.
- **순수 도메인 계층** → 프레임워크에 안 묶여 TDD가 쉬움(핵심 규칙을 빠르게 검증).
